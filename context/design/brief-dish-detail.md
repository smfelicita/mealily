# ТЗ для claude.ai/design — DishDetailPage (редизайн)

Используется как промпт в claude.ai → артефакт React. Цель — исследовать новый визуал детальной страницы блюда для мобильного PWA. Результат — одиночный `.jsx` артефакт с моковыми данными.

---

## Промпт (скопировать в claude.ai/design)

Создай React-артефакт (`.jsx`) — детальная страница блюда для мобильного PWA "Meality". Это редизайн существующего экрана, нужно несколько свежих визуальных решений в рамках заданной дизайн-системы.

### Контекст продукта

Meality — сервис выбора блюд с ИИ-помощником. Пользователь ищет, что приготовить из того, что есть в холодильнике. На детальной странице он смотрит: из чего блюдо, сколько готовить, шаги рецепта, КБЖУ, комментарии, похожие блюда. Основной сценарий: «открыл — прочитал — добавил в план готовки».

### Формат и ограничения

- **Мобильный first** — дизайнь под ширину 375–430px. Layout должен быть отдельным экраном (не модалка), контент скроллится вертикально.
- **Одиночный .jsx файл**, моковые данные внутри компонента.
- Только core-классы Tailwind (без кастомного `tailwind.config`).
- Без `localStorage`/`sessionStorage` — только `useState`.
- Иконки — `lucide-react` (ChevronLeft, Heart, Share2, MoreVertical, Clock, Users, ChefHat, Flame, CalendarPlus, MessageCircle, Refrigerator, Edit, Trash2).

### Дизайн-система (приблизить classes вручную)

Базовый стиль — тёплый, «домашний», но чистый. Не айосный и не материал.

```
Фон страницы:   #F6F4EF  (тёплый беж)
Фон карточек:   #FFFFFF
Фон-3 (мягкий): #F5EFE6
Бордер:         #E5D8C8
Акцент:         #C4704A  (терракотовый)
Акцент-muted:   rgba(196,112,74,0.1)
Sage (успех):   #5C7A59  (мягкий оливковый, используется для "есть в холодильнике")
Text:           #1C1917  (почти чёрный)
Text-2:         #78716C  (вторичный)
Text-3:         #A8A29E  (подсказки)
Шрифт:          Nunito (используй font-sans через Tailwind)
Радиусы:        по умолчанию 16px, полный — 9999px
```

Тени мягкие: `shadow-sm` для карточек, `shadow-lg` для FAB.

### Структура экрана (сверху вниз)

1. **Hero** — горизонтально-свайпаемые фото на высоту ~280–320px. Градиент снизу для читаемости. Оверлей-хедер: «Назад», справа — «Поделиться», «В избранное» (heart), меню (три точки). Индикатор текущего фото (точки или счётчик `1/3`).

2. **Заголовок и описание** — крупный title (~24–28px, bold), ниже description серым цветом text-2.

3. **Мета-полоска** — строка с иконками: время (30 мин), порции (4), сложность (Легко), кухня (Итальянская). Спокойные "chip" или просто icon + label с разделителями.

4. **Автор** — мелкая строка «Рецепт от Марина» с аватаром-инициалом. Если isOwner — показать подпись «Ваш рецепт».

5. **Ингредиенты** — секция с заголовком «Ингредиенты» (17px, semibold) + переключатель справа: иконка-холодильник (когда активна — продукты, которых нет, подсвечены, есть в холодильнике — отмечены check / sage цветом). Список — строки `название ··· количество` (dot-leader или разрежённая сетка). Пункты с `inFridge:true` — бледная sage-галочка слева или sage-текст. С `inFridge:false` — акцентная точка. `basic:true` (соль, перец) — без выделения.

6. **Пошаговый рецепт** — заголовок «Приготовление». Карточки шагов: слева большая цифра (24–32px) в акцентном цвете или muted-круге, справа текст шага. Между шагами — воздух, но без агрессивных divider-ов.

7. **КБЖУ** — «Пищевая ценность» + приписка "на порцию". Четыре плитки в ряд: Калории / Белки / Жиры / Углеводы. Плитка — белая с тонким бордером, округлая, значение крупно (16–18px tabular-nums), единица и подпись ниже серым. Можно попробовать визуализацию — мини-прогресс или цветные точки у каждой плитки.

8. **Теги и категории** — клик-pill'ы: категории (Обед, Ужин) в одном стиле, теги с префиксом `#` — чуть другой вид.

9. **Похожие блюда** («Попробуй также») — горизонтальный скролл-лист из 3 карточек с фото + название + время. Карточки шириной ~160–200px.

10. **Комментарии** — секция в конце: аватар-инициал + имя + текст + timestamp. Pinned-комментарий — лёгкая акцент-плашка сверху или pin-иконка. Input для нового комментария — внизу секции (plain, без излишеств).

11. **FAB** — floating-кнопка «Добавить в план» справа снизу. Акцентный цвет, иконка CalendarPlus, тень. Не перекрывать контент — отступ снизу в скроллере.

### Что важно

- Иерархия: hero → заголовок → мета → ингредиенты → шаги → КБЖУ → теги → похожие → комментарии → FAB. Порядок менять не нужно, можно экспериментировать с визуалом каждой секции.
- Секции разделены мягко: тонкой линией `#E5D8C8` или просто воздухом. Не использовать жёсткие чёрные линии.
- Весь текст — русский.
- Размер секций должен хорошо смотреться именно на мобильном — не лепить плотно, давать дыхание.
- Число шагов — 6, значит карточки должны хорошо выглядеть и при 3, и при 10 шагах.
- Состояния: «в избранном» (heart заполнен акцентом) и «не в избранном» (outline). По клику — toggle.

### Что попросить у claude.ai в первой итерации

Сделай **три визуальных варианта** секции ингредиентов в одном артефакте (показать варианты друг под другом с подписями "Вариант А / Б / В"), остальные секции — в одном каноничном виде. После того как я выберу вариант ингредиентов, будем итерировать остальные секции.

Варианты ингредиентов для сравнения:
- **А** — dot-leader между названием и количеством, статус «в холодильнике» — маленькая sage-точка слева.
- **Б** — двухколоночная сетка с чек-боксами слева (отмеченные — sage, не отмеченные — outline), количество справа мелким серым.
- **В** — карточки-строки с бэкграундом: `in fridge` — bg-bg-3 с тонкой sage-границей, `не in fridge` — bg-white с акцент-точкой.

### Технические напоминания

- `export default` для компонента.
- Никаких вызовов API, всё на моке.
- Все изображения — через Unsplash-ссылки из мока (рабочие).
- Не использовать `<a href="...">` — только кнопки и `onClick` с console.log для действий.
- Избегать библиотечных зависимостей кроме `react` и `lucide-react`.
