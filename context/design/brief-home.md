# ТЗ для claude.ai/design — HomePage

Главная страница Meality. После открытия приложения. Отвечает на вопрос «Что приготовить сегодня?».

---

## Промпт для claude.ai/design

Создай React-артефакт (`.jsx`) — главная страница мобильного PWA Meality. Это редизайн, нужно переосмыслить существующий экран в рамках нашей дизайн-системы.

### Контекст

Пользователь зашёл в приложение. Цель — быстро решить что готовить. Сценарий: «открыл → выбрал время еды (завтрак/обед/ужин/перекус) → посмотрел карточки блюд → либо открыл детальную, либо тапнул "Буду готовить сегодня"».

Под viewport должно помещаться столько карточек блюд, сколько помещается — без скролла. Если пользователь новый (гость) — он видит приветственный баннер с CTA на регистрацию. Если залогинен — видит быстрые действия "Избранное" и "Холодильник".

### Ограничения

- Мобильный first (375–430px).
- Одиночный `.jsx`, моки внутри.
- Tailwind core-классы + hex в style (ваш config не подгружается).
- Иконки — `lucide-react`: ChevronRight, Heart, Refrigerator, Sun, Utensils, Moon, Cookie, Plus, Clock, Flame, Sparkles.
- `useState`, без `localStorage`.

### Дизайн-система (reference — копия из design-system.md)

**Цвета:**
- Фон: `#F6F4EF`
- Карточки: `#FFFFFF`
- Фон мягкий: `#F5EFE6`
- Бордер: `#E5D8C8`
- Акцент: `#C4704A`, accent-muted `rgba(196,112,74,0.1)`
- Sage: `#5C7A59`, sage-muted `rgba(92,122,89,0.08)`
- Text: `#1C1917`, text-2 `#78716C`, text-3 `#A8A29E`
- Шрифт — Nunito. `font-sans` + `fontFamily: 'Nunito, system-ui, sans-serif'`.

**Радиусы:** pill (`rounded-full`) для кнопок и chip'ов, `rounded-2xl` (16px) или `rounded-[14px]` для карточек, `rounded-xl` (12px) для sub-блоков.

**Типографика:** H1 страницы = `text-[26px] font-extrabold leading-tight`. H2 секций = `text-[17px] font-bold`. Body 14–15px. Мета-лейблы 12px text-text-3. 2xs-лейблы uppercase = 10–11px `font-semibold tracking-wide`.

**Кнопки:** все pill. Primary = `bg-accent text-white h-11 px-5 rounded-full`. Secondary = `bg-white text-text border border-border h-11 px-5 rounded-full`. Ghost = прозрачный text-text-2.

**Chip:** primary-active = `bg-accent-muted border border-accent/25 text-accent font-bold`, neutral = `bg-white border border-border text-text-2`, success = `bg-sage/12 border border-sage/30 text-sage`. Базовые classes: `px-3 py-1.5 rounded-full text-[12.5px]`.

**Отступы:** горизонтально `px-5`, между секциями `mt-7` / `mt-8`.

**Тени:** мягкая — `0 1px 6px rgba(0,0,0,0.06)`, accent для FAB — `0 8px 24px rgba(196,112,74,0.45)`.

### Состояния экрана

Сделай в одном артефакте **три варианта сверху вниз** с разделителями-подписями:

1. **Гость (не залогинен)** — приветственный баннер + подборка блюд (показываем даже гостям, чтобы заинтересовать)
2. **Юзер, холодильник пустой** — hint-банер «Заполни холодильник» + обычный список
3. **Юзер, холодильник полон** — нормальный рабочий вид с быстрыми действиями

### Структура экрана (сверху вниз)

1. **Header (shared, показать сверху артефакта, но помнить что в реальности это Layout):** лого + «Моя кухня» слева, справа — колокольчик + аватар-инициал (26px). Можно просто нарисовать как напоминалку — в детальном дизайне не участвует.

2. **H1:** `Что приготовить сегодня?` (две строки на мобильном, `leading-tight`). Можно под него добавить мелкое приветствие типа «Добрый день, Марина 👋» (если залогинен) — 13px text-text-2.

3. **Guest баннер (только вариант 1):** белая карточка с заголовком «Добавь свои блюда — и больше не думай, что готовить», мелкий text-3 подзаголовок, крестик в углу, две кнопки (primary «Создать свою кухню» + ghost «Уже есть аккаунт? Войти»).

4. **Мета-полоска дня (новое, фишка системы):** 3 мини-метрики в белой карточке — например «В плане на сегодня: 2», «В холодильнике: 12», «Избранных: 8». Если гость — показать другое (например «Популярные блюда» / «~300 рецептов» / «Готовится за 5 мин»). Стиль как MetaStrip из design-system.

5. **MealType чипы:** 4 кнопки (Завтрак / Обед / Ужин / Перекус) в ряд, с иконкой lucide слева. Активный = accent-muted с бордером accent/25. По умолчанию активен текущий прием пищи (для демо — BREAKFAST). Можно добавить чип «Все» впереди.

6. **Sage-плашка-подсказка (для варианта 3):** «Из вашего холодильника можно приготовить 7 блюд» — sage-muted фон, sage-текст, Refrigerator иконка. Клик → фильтр «только из холодильника».

7. **Список блюд:** карточки в строку (row-variant), 76-90px высота. Слева фото 64–72px rounded-xl, справа — название, мета-строка (время + калории + mealTime-chip), справа внизу действия (Heart, Plus-в-план). Если `missingCount > 0` — мелкий accent-текст «докупить 2» или accent-точка слева. Если `inFridge:true` — крошечная sage-галочка у времени.

8. **Pinned-блок (новое, фишка системы):** один «особенный» блок «Сегодня в плане» — accent-muted фон с accent-border, внутри мини-карточка блюда + кнопка «Готовлю!». Показывается только если на сегодня что-то запланировано.

9. **Кнопка "+ Добавить своё блюдо":** белая pill во всю ширину с accent-текстом. На её месте в гостевом режиме — CTA на регистрацию.

10. **Quick actions (только вариант 3, залогинен):** 2 pill-кнопки в ряд — «❤ Избранное» и «🧊 Холодильник» — активные = accent-фон, неактивные = sage-фон. (Sage намеренно другой от артефакта, т.к. это постоянные действия а не tab-фильтр).

### Фишки для этой страницы

- **Мета-полоска с 3 личными метриками** — новое, раньше не было. Делает экран «своим».
- **Pinned-блок «Сегодня в плане»** — видимое приглашение готовить. Если плана нет — скрываем блок.
- **Приветствие с именем пользователя** — мелкая «душевная» деталь.
- **Sage-плашка про холодильник** — показывает ценность фичи, которую пользователь мог не заметить.

### Что попросить у артефакта дополнительно

В конце артефакта добавь маленький комментарий-подпись с размером viewport и стрелками скролла если список блюд не помещается. Это чтобы видеть, помещаются ли 4 блюда в экран или нужно сокращать карточки.

### Технические напоминания

- `export default`
- Никаких API / `localStorage`
- Все 3 варианта в одном артефакте друг под другом, с подписью-разделителем «— Вариант 1: гость —», «— Вариант 2: юзер, пустой холодильник —», «— Вариант 3: рабочий вид —»
- Между вариантами — зазор `h-8 bg-gray-200` с подписью по центру
- Шрифт — загрузить Nunito через `<link>` в `<head>` я не смогу, просто укажи в style `fontFamily: 'Nunito, system-ui, sans-serif'` на корневом div и на всём унаследуется
