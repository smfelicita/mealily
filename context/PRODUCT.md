# Meality — Продуктовый документ

---

## 1. Что это за продукт

Meality — сервис для планирования питания с ИИ-помощником.
Помогает решить ежедневный вопрос "что приготовить" — на основе того,
что уже есть дома, времени суток и личных предпочтений.

Доступен как веб-приложение (PWA, устанавливается на телефон) и Telegram-бот.

---

## 2. Проблемы которые решаем

| Проблема | Решение |
|---|---|
| "Не знаю что приготовить" | ИИ-помощник и подбор блюд по времени дня |
| "Надо идти в магазин, но не знаю за чем" | Холодильник + блюда только из того что есть |
| "Хочу хранить свои рецепты" | Добавление своих блюд |
| "Хочу готовить вместе с семьёй/друзьями" | Группы с общими рецептами |
| "Некогда открывать приложение" | Telegram-бот всегда под рукой |

---

## 3. Целевая аудитория

- Люди которые готовят дома и не знают что приготовить
- Семьи которые хотят планировать питание вместе
- Те кто хочет готовить из того что уже есть в холодильнике

---

## 4. Роли пользователей

### Гость (не авторизован)
- Просматривает базу рецептов
- Ищет и фильтрует блюда
- Пользуется ИИ-помощником (лимит: 2 сообщения в день)
- При попытке открыть Холодильник → модальное окно "Зарегистрируйтесь"
- При исчерпании лимита ИИ → предложение зарегистрироваться

### User (авторизованный, бесплатно)
- Всё что может Гость — без ограничений
- Холодильник: добавлять/удалять продукты
- ИИ-помощник знает что есть в холодильнике
- Добавлять свои рецепты (по умолчанию видны группе)
- Создавать группы и приглашать участников
- Вступать в группы по ссылке

### Pro (авторизованный, платная подписка) — на перспективу
- Всё что может User
- Добавление продуктов в холодильник по фото чека из магазина
- Другие премиум-функции (TBD)

### Admin
- Управление базой блюд (добавить/редактировать/удалить)
- Модерация рецептов пользователей
- Управление пользователями

## 5. Функционал

### 5.1 Рецепты и блюда

**Карточка блюда содержит:**
- Название (рус + англ)
- Фото (загружается файлом)
- Категория: завтрак / обед / ужин / перекус / суп / салат / десерт / напиток
- Теги: вегетарианское, быстро (до 30 мин), без глютена, острое и т.д.
- Время приготовления (минуты)
- Сложность: легко / средне / сложно
- Калории
- **Список ингредиентов с количеством** (напр. "Яйца — 3 шт", "Мука — 200г")
- Пошаговый рецепт (текст, markdown)
- Видео (загружается файлом или ссылка на YouTube)
- Автор (null = системное блюдо от Meality)

**Видимость рецепта:**
- Системные блюда → публичные (видны всем включая гостей)
- Рецепт пользователя → по умолчанию виден его группам
- Пользователь может вручную сделать рецепт публичным

**Каталог блюд:**
- Поиск по названию и ингредиентам
- Фильтр по категории, тегам, сложности, времени готовки
- Фильтр по времени суток (утро/день/вечер)
- Режим "из холодильника" — только блюда из имеющихся продуктов
- Сортировка (по умолчанию, по калориям, по времени)

### 5.2 Холодильник

- Список продуктов которые есть у пользователя
- **Способ 1:** ввод текстом — пишет название, система распознаёт из базы
- **Способ 2:** выбор из готового списка (поиск + чекбоксы)
- Удалить продукты
- Дата добавления, срок годности (опционально)
- Кнопка "Показать блюда из холодильника"
- База продуктов: расширить до 100-200 наименований
- **Pro:** добавить продукты по фото чека из магазина
- Telegram-бот: управление холодильником временно не используем

### 5.3 ИИ-помощник

- Чат с Claude AI
- Контекст: что есть в холодильнике, список доступных блюд
- Может порекомендовать блюдо, ответить на вопрос о рецепте
- Быстрые кнопки-подсказки (что на завтрак? что быстро?)
- История чата сохраняется (только для авторизованных)
- Для гостей: лимит 2 сообщения в день (по сессии), история не сохраняется
- При исчерпании лимита → предложение зарегистрироваться

### 5.4 Группы

- Создать группу (название, описание, аватар)
- Пригласить участников по ссылке или email
- Роли в группе: owner / member
- Рецепты всех участников группы видны другим участникам
- Несколько групп у одного пользователя (например: семья, друзья)

### 5.5 Профиль пользователя

- Имя, email, аватар
- Мои рецепты
- Мои группы
- Подписка (статус, дата окончания)
- Привязка Telegram-аккаунта

### 5.6 Telegram-бот

- Те же функции что в веб-приложении
- Привязка к аккаунту на сайте через команду /connect
- Команды: /start, /breakfast, /lunch, /dinner, /snack, /fridge, /random, /chat
- Управление холодильником
- ИИ-чат
- Просмотр рецептов и рецепта
- Лимиты для незарегистрированных пользователей бота

## 6. Пути пользователей (User Journeys)

### 6.1 Гость — первый визит

```
Заходит на сайт
  → Видит главную с блюдами по времени дня
  → Листает каталог, кликает на блюдо
  → Видит рецепт, ингредиенты, фото/видео
  → Кликает "Холодильник" → модальное окно "Войдите или зарегистрируйтесь"
  → Пишет в ИИ-чат → получает ответ (счётчик -1 из 5)
  → После 2 сообщения → "Зарегистрируйтесь чтобы продолжить"
  → Регистрируется → становится User
```

### 6.2 User — ежедневное использование

```
Открывает приложение (PWA на телефоне) или Telegram-бот
  → Видит блюда для текущего времени суток
  → Включает режим "из холодильника"
  → Видит блюда которые можно приготовить прямо сейчас
  → Выбирает блюдо → смотрит рецепт
  → Или: пишет в ИИ "что приготовить на обед если есть яйца и сыр?"
  → ИИ предлагает варианты с рецептами
```

### 6.3 User — добавляет свой рецепт

```
Нажимает "Добавить рецепт"
  → Заполняет форму (название, ингредиенты с количеством, шаги)
  → Загружает фото
  → Добавляет видео (ссылка или загрузка)
  → Выбирает видимость (моя группа / публично)
  → Сохраняет → рецепт доступен в его группах
```

### 6.4 User — создаёт группу

```
Идёт в раздел Группы
  → Создаёт группу "Наша семья"
  → Копирует ссылку-приглашение
  → Отправляет ссылку в мессенджер
  → Участники переходят по ссылке → вступают в группу
  → Теперь все видят рецепты друг друга
```

### 6.5 Telegram-бот — утренний сценарий

```
Пользователь пишет /breakfast или нажимает кнопку "Завтрак"
  → Бот показывает 3 варианта завтрака из холодильника
  → Нажимает на блюдо → бот показывает рецепт
  → Или пишет в свободной форме → ИИ отвечает
```

## 7. Технический стек

- **Бэкенд:** Node.js + Express + Prisma ORM
- **База данных:** локальная PostgreSQL на прод-сервере (Москва)
- **Фронтенд:** React + Vite (PWA)
- **Telegram:** node-telegram-bot-api
- **ИИ:** Claude API (Anthropic)
- **Медиафайлы:** локальный диск + nginx (`STORAGE_DRIVER=local`)
- **Хостинг:** прод-сервер 5.42.112.233 (Москва) — бэкенд + фронтенд + бот + БД
