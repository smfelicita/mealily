# Meality — Frontend Standards

_Актуально на апрель 2026_

## Общие принципы
- mobile-first
- все экраны собираются из компонентов
- не создавать дубли компонентов
- не добавлять новые UI-паттерны без необходимости
- один компонент = одна ответственность

## Стек
- React + Vite
- TailwindCSS
- Zustand (state)
- React Router v6

## Структура проекта

```
/src
  /pages           — экраны (один файл = один экран)
  /components
    /ui            — переиспользуемые UI без бизнес-логики
    /domain        — доменные компоненты с логикой
  /hooks           — кастомные хуки
  /lib             — утилиты, api, store
  /constants       — константы
```

## Компоненты

**UI-компоненты** (`/components/ui`):
- переиспользуемые
- без бизнес-логики
- принимают данные и колбэки через props

**Доменные компоненты** (`/components/domain`):
- логика приложения
- используют UI-компоненты
- могут обращаться к store / api

## Naming
- компоненты — PascalCase
- props / функции — camelCase
- файлы компонентов — PascalCase.jsx

## Tailwind правила

**Layout:**
- страницы → `flex flex-col` / `grid`
- внутренние блоки → `flex`

**Spacing:**
- использовать только стандартную шкалу Tailwind (4, 6, 8, 12, 16...)
- не писать случайные px

**Запрещено:**
- inline styles (`style={{...}}`)
- дубли классов
- хаотичные комбинации

## Layout
- контейнер: `max-w-md mx-auto px-4`
- вертикальные отступы: `gap-4` / `gap-6`
- не использовать фиксированные высоты без необходимости

## Карточки
- один компонент: `RecipeCard`
- используется во всех списках
- не создавать альтернативные версии

## Списки
- `RecipeList` — единый компонент
- не дублировать списки под разные экраны

## Формы
- единый стиль: label → input → error
- обязательные поля помечаются `*`
- ошибки отображаются под полем

## Состояния
Каждый экран и компонент должен иметь:
- `default`
- `empty`
- `loading`
- `error`

## Модалки
- использовать компонент `Modal`
- не создавать кастомные оверлеи

## Навигация
- `BottomTabBar` — фиксированный
- `TopBar` — одинаковый на всех экранах

## Поведение
- кликабельные элементы — `<button>` или `<Link>`
- не использовать `<div>` как кнопку
- focus states обязательны (`focus:outline-none focus:ring-2`)

## Правила продукта
- Recipes — это мои / семейные блюда
- Избранное — фильтр внутри Recipes, не отдельный tab
- Готовые рецепты — только для копирования (шаблоны), не главный каталог
- Home — экран принятия решения "что приготовить сегодня"

## Порядок работы
1. Перечислить компоненты которые будут использованы
2. Показать структуру файлов
3. Написать код
4. Если задача большая — разбивать на шаги

## Запрещено
- дубли компонентов
- разные версии одного и того же UI
- быстрая вёрстка на месте
- нарушение структуры проекта
